Detailed Description

Configures the SDK's Big Segments behavior.

Not thread-safe. Construct, configure, and call Build on a single thread; the resulting built::BigSegmentsConfig is safe to share.

◆ ContextCacheSize()

BigSegmentsBuilder & launchdarkly::server_side::config::builders::BigSegmentsBuilder::ContextCacheSize ( std::size_t  size)

Sets the maximum number of context membership lookups cached by the SDK. Defaults to 1000.

To reduce store traffic, the SDK maintains an LRU cache keyed by context key. A higher value reduces store queries for recently-referenced contexts at the cost of memory.

◆ ContextCacheTime()

BigSegmentsBuilder & launchdarkly::server_side::config::builders::BigSegmentsBuilder::ContextCacheTime ( std::chrono::milliseconds  ttl)

Sets the time-to-live for cached membership lookups. Defaults to 5 seconds.

A higher value reduces store queries for any given context, but delays the SDK noticing membership changes. Zero or negative durations are coerced to the default.

◆ StaleAfter()

BigSegmentsBuilder & launchdarkly::server_side::config::builders::BigSegmentsBuilder::StaleAfter ( std::chrono::milliseconds  threshold)

Sets how long the SDK waits before treating store data as stale. Defaults to 2 minutes.

If the store's last-updated timestamp falls behind the current time by more than this duration, evaluations report a big segments status of STALE and the status provider reports the store as stale. Zero or negative durations are coerced to the default.

◆ StatusPollInterval()

BigSegmentsBuilder & launchdarkly::server_side::config::builders::BigSegmentsBuilder::StatusPollInterval ( std::chrono::milliseconds  interval)

Sets the interval at which the SDK polls the store's metadata to determine availability and staleness. Defaults to 5 seconds.

Zero or negative durations are coerced to the default.
