\doxysubsection{Detailed Description}
\mbox{\hyperlink{classlaunchdarkly_1_1server__side_1_1data__systems_1_1FDv2DataSystem}{FDv2\+Data\+System}} is the IData\+System implementation for the FDv2 protocol. It runs a sequence of initializers to populate an in-\/memory store, then hands off to a synchronizer for ongoing updates.

Lifecycle / call order\+:


\begin{DoxyEnumerate}
\item Construct.
\item Call \mbox{\hyperlink{classlaunchdarkly_1_1server__side_1_1data__systems_1_1FDv2DataSystem_a5ec46f578c9a24eb00a234a21e2d7d9d}{Initialize()}} exactly once. It returns immediately; orchestration runs asynchronously on the executor.
\item Call IStore methods (Get\+Flag/\+Get\+Segment/\+All\+Flags/\+All\+Segments) and \mbox{\hyperlink{classlaunchdarkly_1_1server__side_1_1data__systems_1_1FDv2DataSystem_a1e8490bb24fbc0ba667767da6a5a3a9d}{Initialized()}} / \mbox{\hyperlink{classlaunchdarkly_1_1server__side_1_1data__systems_1_1FDv2DataSystem_a55601f96aa95fe825d918babecee5433}{Identity()}} any number of times, from any thread.
\item Drain the executor and join callers (see Destruction protocol).
\item Destroy.
\end{DoxyEnumerate}

\mbox{\hyperlink{classlaunchdarkly_1_1server__side_1_1data__systems_1_1FDv2DataSystem_a5ec46f578c9a24eb00a234a21e2d7d9d}{Initialize()}} may not be called more than once.

Thread safety\+:


\begin{DoxyItemize}
\item Get\+Flag, Get\+Segment, All\+Flags, All\+Segments, Initialized, and Identity are safe to call concurrently from any thread.
\item \mbox{\hyperlink{classlaunchdarkly_1_1server__side_1_1data__systems_1_1FDv2DataSystem_a5ec46f578c9a24eb00a234a21e2d7d9d}{Initialize()}} is intended to be called once from a single thread.
\item The destructor must run with no other method calls in flight; see Destruction protocol below.
\end{DoxyItemize}

Destruction protocol\+:

The destructor cancels in-\/flight orchestration (closes the active source) but does NOT block to drain executor callbacks that may already be queued. Before destroying, the caller must ensure both of\+:


\begin{DoxyEnumerate}
\item The executor that orchestration callbacks run on has been stopped AND any thread running it has been joined. Otherwise a previously-\/ scheduled callback may run and reference a destroyed object.
\item No other thread is currently calling a method on this object.
\end{DoxyEnumerate}

The standard pattern is\+: \begin{DoxyVerb}ioc.stop();          // no new callbacks; running work returns
run_thread.join();   // wait for the executor thread to exit
// Now no thread can be touching this object.
// Destroy FDv2DataSystem.
\end{DoxyVerb}
 \mbox{\hyperlink{classlaunchdarkly_1_1server__side_1_1ClientImpl}{Client\+Impl}}\textquotesingle{}s destructor uses this pattern ($\sim$\+Client\+Impl performs ioc\+\_\+.\+stop() and run\+\_\+thread\+\_\+.\+join() before any member is destroyed).

Sources (initializers and synchronizers)\+:

Sources are constructed lazily via the factories supplied at construction. Each factory is invoked at most once during a run of the orchestration. A source is closed and destroyed when the orchestration finishes processing a result that ends its turn, or when the \mbox{\hyperlink{classlaunchdarkly_1_1server__side_1_1data__systems_1_1FDv2DataSystem}{FDv2\+Data\+System}} is destroyed. At any given moment at most one source is active.

Selector tracking\+:

\mbox{\hyperlink{classlaunchdarkly_1_1server__side_1_1data__systems_1_1FDv2DataSystem}{FDv2\+Data\+System}} tracks the most-\/recent non-\/empty selector seen on a Change\+Set and passes it into each subsequent synchronizer Next() call. The initial selector is empty.

Orchestration phases\+: \begin{DoxyVerb}      Initialize()
          |
          v
+-------------------+   no factories given
|  Offline?         |---------> [Done, status = kValid]
+-------------------+
          |
          v
+-------------------+    initializer #N returns:
|  Initializer phase|      ChangeSet(no selector) -> stay, N += 1
|  N = 0, 1, 2, ... |      ChangeSet(selector)    -> go to Sync
|                   |      Interrupted/Terminal   -> stay, N += 1
|                   |      Goodbye                -> stay, N += 1
|                   |      Shutdown               -> [Closed]
+-------------------+
          |
          | (N exhausted, or basis received)
          v
+-------------------+    active synchronizer's Next returns:
|  Synchronizer     |      ChangeSet      -> apply, loop
|  phase            |      Interrupted    -> loop (source self-retries)
|  (cyclic;         |      Goodbye        -> loop (source self-restarts)
|   blocked sources |      TerminalError  -> block, advance
|   are skipped)    |      Shutdown       -> [Closed]
+-------------------+
              ^   |     fallback condition  -> advance (with wrap)
              |   |     recovery condition  -> reset to first available
              +---+
          |
          | (all synchronizers blocked)
          v
[Done; final status preserved]
\end{DoxyVerb}
 Status transitions\+:

k\+Initializing (initial) -\/$>$ k\+Valid on first successful Change\+Set apply. k\+Interrupted on Interrupted / Terminal\+Error (filtered to k\+Initializing while still in the initializer phase if not yet Valid). k\+Off if all initializers exhaust without data and no synchronizers are configured. k\+Valid -\/$>$ k\+Interrupted on errors; k\+Off when synchronizers cycle through and exhaust. k\+Interrupted -\/$>$ k\+Valid on next successful Change\+Set apply; k\+Off on synchronizer exhaustion. k\+Off -\/$>$ terminal.
