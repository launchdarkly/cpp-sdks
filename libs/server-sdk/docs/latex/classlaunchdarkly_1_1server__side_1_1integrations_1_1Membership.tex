\doxysubsection{Detailed Description}
\mbox{\hyperlink{classlaunchdarkly_1_1server__side_1_1integrations_1_1Membership}{Membership}} describes which Big Segments a single context belongs to, as reported by an \mbox{\hyperlink{classlaunchdarkly_1_1server__side_1_1integrations_1_1IBigSegmentStore}{IBig\+Segment\+Store}} lookup. 

Implementations of \mbox{\hyperlink{classlaunchdarkly_1_1server__side_1_1integrations_1_1IBigSegmentStore}{IBig\+Segment\+Store}} should construct a \mbox{\hyperlink{classlaunchdarkly_1_1server__side_1_1integrations_1_1Membership}{Membership}} using \mbox{\hyperlink{classlaunchdarkly_1_1server__side_1_1integrations_1_1Membership_a08eeda4ab194e704cfadab4da731746b}{Membership\+::\+From\+Segment\+Refs}} and return it from \mbox{\hyperlink{classlaunchdarkly_1_1server__side_1_1integrations_1_1IBigSegmentStore_aa32a6f78d62ee3e67d9ac9cdcec7a1a8}{IBig\+Segment\+Store\+::\+Get\+Membership}}. The returned \mbox{\hyperlink{classlaunchdarkly_1_1server__side_1_1integrations_1_1Membership}{Membership}} is an immutable snapshot — once constructed it does not observe any later changes to the underlying store.

A \char`\"{}segment ref\char`\"{} is the string {\ttfamily $<$segment\+Key$>$.g$<$generation$>$}; the SDK constructs that string when evaluating a flag and passes it to \mbox{\hyperlink{classlaunchdarkly_1_1server__side_1_1integrations_1_1Membership_acab81b15620e298fea8f6638bc726caa}{Check\+Membership}}.

Implemented inline because integration libraries link against the server-\/sdk shared library, which only exports the C API — out-\/of-\/line symbols defined here would not be visible to consumers.
