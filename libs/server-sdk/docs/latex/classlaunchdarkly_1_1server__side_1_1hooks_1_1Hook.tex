\doxysubsection{Detailed Description}
Base interface for hook implementations.

All stage methods have default implementations that take no action, allowing hook implementations to only override the stages they need.

This interface is designed for forward compatibility -\/ new stages can be added without breaking existing hook implementations.

IMPORTANT LIFETIME CONSIDERATIONS\+:
\begin{DoxyItemize}
\item All context objects passed to hook stages are valid only during the execution of that stage.
\item Do not store references, pointers, or string\+\_\+views from context objects.
\item If you need data beyond the immediate execution of a stage, copy it to owned types (std\+::string, \mbox{\hyperlink{classlaunchdarkly_1_1Value}{Value}}, etc.).
\item \mbox{\hyperlink{classlaunchdarkly_1_1server__side_1_1hooks_1_1EvaluationSeriesData}{Evaluation\+Series\+Data}} should only be used within the stage or returned to be passed to the next stage. 
\end{DoxyItemize}
