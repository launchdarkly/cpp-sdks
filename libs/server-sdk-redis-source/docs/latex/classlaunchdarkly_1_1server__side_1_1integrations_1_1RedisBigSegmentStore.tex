\doxysubsection{Detailed Description}
\mbox{\hyperlink{classlaunchdarkly_1_1server__side_1_1integrations_1_1RedisBigSegmentStore}{Redis\+Big\+Segment\+Store}} is a Big Segments persistent store backed by Redis. 

Call \mbox{\hyperlink{classlaunchdarkly_1_1server__side_1_1integrations_1_1RedisBigSegmentStore_af13f3d347d2b7752b198f44e40cf4f81}{Redis\+Big\+Segment\+Store\+::\+Create}} to obtain a new instance, then pass it to the SDK via the Big Segments config builder.

The same Redis database can be shared with \mbox{\hyperlink{classlaunchdarkly_1_1server__side_1_1integrations_1_1RedisDataSource}{Redis\+Data\+Source}} — Big Segments keys use their own prefixed namespaces ({\ttfamily big\+\_\+segment\+\_\+include}, {\ttfamily big\+\_\+segment\+\_\+exclude}, {\ttfamily big\+\_\+segments\+\_\+synchronized\+\_\+on}) and do not conflict with the flag/segment hashes. The Launch\+Darkly Relay Proxy is responsible for populating Big Segments data in Redis; this class only reads from it.

This implementation is backed by \href{https://github.com/sewenew/redis-plus-plus}{\texttt{ Redis++}}, a C++ wrapper for the \href{https://github.com/redis/hiredis}{\texttt{ hiredis}} library.
